#------- Reflection Task --------
RAG stands for Retrieval-Augmented Generation. It is a method where relevant pieces of information are retrieved from a dataset and then sends only that information to a language model like Gemini to generate an answer.

The context window is the limited amount of text an LLM can process at one time. We cannot always send a whole book, website, or dataset to an LLM because models have input size limits. Even if it were possible, it would be inefficient and expensive, which can reduce the quality of the answer.

Iterators and yield help us handle large files efficiently by streaming data one piece at a time instead of loading everything at once into memory. This allows to process large datasets gradually and stop early once enough relevant information is found.

One key trade-off is between completeness and noise. If too little context is included, the model could miss important information and give an incomplete answer. However, if too much context is included, the prompt becomes noisy, slower, and more expensive, which can also reduce answer quality.

For example, with 50 lecture transcripts, a simple RAG system can read through each file one by one, select chunks that contain keywords such as “dynamic programming,” and include only those chunks in the prompt. Gemini then uses this selected context to answer the question.
